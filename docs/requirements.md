# 案件管理システム 要件定義書

## 1. プロジェクト概要

### 1.1 システム名
案件管理システム（Project Management System）

### 1.2 目的
社内の案件（プロジェクト）情報を一元管理し、進捗状況の可視化・担当者間の情報共有を効率化する。

### 1.3 対象ユーザー
- 管理者（Admin）：システム全体の設定・ユーザー管理を行う
- マネージャー（Manager）：案件の作成・編集・担当者アサインを行う
- メンバー（Member）：担当案件の閲覧・ステータス更新を行う

### 1.4 技術スタック（想定）
- フロントエンド：Next.js (App Router) + TypeScript
- バックエンド：Next.js API Routes
- データベース：PostgreSQL（Neon）
- ORM：Prisma
- 認証：NextAuth.js
- デプロイ：Vercel

## 2. 機能要件

### 2.1 認証・ログイン機能

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| AUTH-001 | メールアドレス・パスワードログイン | メールアドレスとパスワードによる認証 | 必須 |
| AUTH-002 | ログアウト | セッションを破棄しログイン画面へ遷移 | 必須 |
| AUTH-003 | パスワードリセット | メール送信によるパスワード再設定 | 必須 |
| AUTH-004 | セッション管理 | JWT/セッションベースの認証状態維持 | 必須 |
| AUTH-005 | ロールベースアクセス制御（RBAC） | Admin / Manager / Member の3ロール | 必須 |
| AUTH-006 | ユーザー登録（管理者のみ） | 管理者がユーザーを招待・登録する | 必須 |

### 2.2 案件管理機能

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| PROJ-001 | 案件一覧表示 | 全案件をリスト/カード形式で表示 | 必須 |
| PROJ-002 | 案件詳細表示 | 個別案件の詳細情報を表示 | 必須 |
| PROJ-003 | 案件新規作成 | 案件名・概要・期間・担当者等を入力し作成 | 必須 |
| PROJ-004 | 案件編集 | 既存案件の情報を更新 | 必須 |
| PROJ-005 | 案件削除（論理削除） | 案件をアーカイブ扱いにする | 必須 |
| PROJ-006 | ステータス管理 | 未着手→進行中→完了→保留 の状態遷移 | 必須 |
| PROJ-007 | 担当者アサイン | 案件に担当者を割り当てる（複数可） | 必須 |
| PROJ-008 | 案件検索・フィルタ | ステータス・担当者・期間等で絞り込み | 重要 |
| PROJ-009 | 案件ソート | 期限・作成日・ステータス等で並び替え | 重要 |

### 2.3 ダッシュボード機能

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| DASH-001 | 案件サマリー表示 | ステータス別の案件数を表示 | 重要 |
| DASH-002 | 自分の担当案件一覧 | ログインユーザーの担当案件を表示 | 必須 |
| DASH-003 | 期限切れアラート | 期限超過・期限間近の案件をハイライト | 重要 |

### 2.4 ユーザー管理機能（管理者向け）

| ID | 機能 | 説明 | 優先度 |
|----|------|------|--------|
| USER-001 | ユーザー一覧表示 | 登録ユーザーの一覧を表示 | 必須 |
| USER-002 | ユーザー追加 | 新規ユーザーを登録する | 必須 |
| USER-003 | ユーザー編集 | ユーザー情報・ロールを変更する | 必須 |
| USER-004 | ユーザー無効化 | アカウントを無効にする（論理削除） | 必須 |

## 3. 非機能要件

### 3.1 セキュリティ
- パスワードはbcryptでハッシュ化して保存する
- CSRF対策を実施する
- SQL Injection対策としてORMを使用する
- HTTPS通信を前提とする
- 認証トークンの有効期限を設定する（デフォルト24時間）

### 3.2 パフォーマンス
- ページ初期表示：2秒以内
- API応答時間：500ms以内（95パーセンタイル）
- 同時接続ユーザー数：50人程度を想定

### 3.3 可用性
- 稼働率：99.5%以上を目標
- Vercel + Neonのマネージドサービスにより高可用性を確保

### 3.4 保守性
- TypeScriptによる型安全なコーディング
- Prismaによるスキーマ管理・マイグレーション
- ESLint + Prettier によるコード品質維持

## 4. データモデル（概要）

### 4.1 User（ユーザー）
| カラム | 型 | 説明 |
|--------|------|------|
| id | UUID | 主キー |
| email | String | メールアドレス（ユニーク） |
| password | String | ハッシュ化パスワード |
| name | String | 表示名 |
| role | Enum | ADMIN / MANAGER / MEMBER |
| isActive | Boolean | 有効フラグ |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |

### 4.2 Project（案件）
| カラム | 型 | 説明 |
|--------|------|------|
| id | UUID | 主キー |
| title | String | 案件名 |
| description | Text | 案件概要 |
| status | Enum | NOT_STARTED / IN_PROGRESS / COMPLETED / ON_HOLD |
| startDate | Date | 開始日 |
| endDate | Date | 終了予定日 |
| createdBy | UUID | 作成者（User外部キー） |
| isArchived | Boolean | アーカイブフラグ |
| createdAt | DateTime | 作成日時 |
| updatedAt | DateTime | 更新日時 |

### 4.3 ProjectAssignment（案件担当）
| カラム | 型 | 説明 |
|--------|------|------|
| id | UUID | 主キー |
| projectId | UUID | 案件ID（外部キー） |
| userId | UUID | ユーザーID（外部キー） |
| assignedAt | DateTime | アサイン日時 |

## 6. API エンドポイント一覧

### 認証
| メソッド | パス | 説明 |
|----------|------|------|
| POST | /api/auth/login | ログイン |
| POST | /api/auth/logout | ログアウト |
| POST | /api/auth/reset-password | パスワードリセット要求 |
| PUT | /api/auth/reset-password/[token] | パスワード再設定 |

### 案件
| メソッド | パス | 説明 |
|----------|------|------|
| GET | /api/projects | 案件一覧取得 |
| GET | /api/projects/[id] | 案件詳細取得 |
| POST | /api/projects | 案件作成 |
| PUT | /api/projects/[id] | 案件更新 |
| DELETE | /api/projects/[id] | 案件削除（論理削除） |
| POST | /api/projects/[id]/assign | 担当者アサイン |
| DELETE | /api/projects/[id]/assign/[userId] | 担当者解除 |

### ユーザー
| メソッド | パス | 説明 |
|----------|------|------|
| GET | /api/users | ユーザー一覧取得 |
| GET | /api/users/[id] | ユーザー詳細取得 |
| POST | /api/users | ユーザー作成 |
| PUT | /api/users/[id] | ユーザー更新 |
| PATCH | /api/users/[id]/deactivate | ユーザー無効化 |

## 7. 開発フェーズ

### Phase 1（MVP）
- ログイン・ログアウト機能
- 案件CRUD（作成・読み取り・更新・削除）
- 案件一覧・詳細表示
- ロールベースアクセス制御

### Phase 2
- ダッシュボード機能
- 案件検索・フィルタ・ソート
- パスワードリセット機能
- 期限切れアラート

### Phase 3（拡張）
- 通知機能（メール/アプリ内）
- コメント機能
- ファイル添付機能
- アクティビティログ
